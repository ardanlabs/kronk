# Release Notes - v1.23.7

**Release Date:** April 24, 2026

## Overview

Version 1.23.7 refactors the Kronk SDK to use a unified option pattern, improves context handling, and introduces new configuration options for cataloger and queue depth.

## Detailed Changes

### New Features

#### Unified Option Handling in Kronk SDK

- Refactored `kronk.New` to use `model.Option` instead of a private `Option` type, making it consistent with the rest of the SDK.
- Added `kronk.NewWithContext` to allow passing a context, enabling better support for trace IDs during model loading.
- Added `WithCataloger` and `WithQueueDepth` options to the model configuration.

### Improvements

#### SDK Refactoring

- Updated `model.NewModel` to retrieve the `Cataloger` from the `Config` struct instead of as a separate argument.

#### Example and Test Updates

- Updated multiple examples and test suites to reflect the changes in the `kronk.New` and `model.Option` APIs.

### Upgrade Notes

#### Breaking Changes

Refactoring `kronk.New` and `model.NewModel` might require updates to existing code that relies on the old signatures.

#### Migration

Existing code using `kronk.New(cfg, ...)` should be updated to use the new `model.Option` based API or `kronk.NewWithContext`.

#### Recommended Actions

- Update your usage of `kronk.New` to the new option-based pattern.
- Leverage `NewWithContext` if you need to pass trace IDs.
