### Upgrade Notes

#### Breaking Changes

The removal of `IgnoreIntegrityCheck` from model configurations is a breaking change for users who relied on this option to bypass integrity checks.

#### Migration

No migration required for most users. If you were using `IgnoreIntegrityCheck`, you will now need to ensure your model files are valid.

#### Recommended Actions

- Ensure all local model files are correct to avoid integrity check failures.
