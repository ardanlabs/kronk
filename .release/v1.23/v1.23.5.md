# Release Notes - v1.23.5

**Release Date:** April 24, 2026

## Overview

Version 1.23.5 introduces a major overhaul of the Incremental Message Cache (IMC) system, moving toward a more efficient unified KV cache architecture and deprecating the older System Prompt Cache (SPC) in favor of a more flexible, RAM-based IMC.

## Detailed Changes

### New Features

#### Unified KV Cache Architecture (#515)

- **Author:** William Kennedy
- Implements a unified KV cache where all slots share a single large pool of tokens.
- This allows idle slots to not reserve capacity, providing better memory utilization.
- Enables KV pressure eviction to reclaim stale slots when the shared pool reaches capacity.
- Decouples the KV state from physical slots by externalizing it to RAM.

#### Enhanced Logging Utilities (#515)

- Added human-readable formatting for bytes (`fmtBytes`) and durations (`fmtDur`) in the model package.

### Improvements

#### Incremental Message Cache (IMC) Overhaul (#515)

- Replaced the older System Prompt Cache (SPC) with a revamped IMC architecture.
- IMC now externalizes KV state to RAM after the cache build, eliminating the need for dedicated cache sequences.
- Updated the `NCtx` calculation to account for `NSeqMax` in unified mode.
- Improved support for agentic workflows by optimizing how conversation history is preserved and restored.

#### Improved Tool Call Parsing (#515)

- Enhanced Qwen model tool-call processing to correctly parse JSON scalars (bool, null, numbers) instead of treating them as strings.

#### Default NSeqMax Adjustment (#512)

- Updated the default `NSeqMax` recommendation in the model analysis tool from 2 to 1 to more accurately reflect unconfigured baseline behaviors.

### Bug Fixes

#### Default Configuration Fix (#512)

- Fixed an issue where the default `NSeqMax` was incorrectly set for analyzing model concurrency.

### Upgrade Notes

#### Breaking Changes

The System Prompt Cache (SPC) has been deprecated and replaced by the improved IMC system. While existing configurations using `incremental_cache: true` should continue to work, the specific `system-prompt-cache` configuration parameter is being phased out.

#### Migration

No manual migration is required for most users. Ensure your `model_config.yaml` is using `incremental_cache: true` to benefit from the new architecture.

#### Recommended Actions

- Review your model configurations to ensure they are optimized for the new unified KV cache.
- Test your agentic workflows (e.g., Cline, OpenCode) to confirm that the improved IMC provides the expected performance gains.
