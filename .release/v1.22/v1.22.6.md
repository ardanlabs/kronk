# Release Notes - v1.22.6

## Summary
This release improves the default `max_tokens` behavior, allowing it to default to the model's context window size when not specified. It also introduces new Tic-Tac-Toe example walkthroughs demonstrating JSON Schema capabilities and adds `presence_penalty` support for certain models.

## Features
- **New Example Walkthroughs**: Added new Tic-Tac-Toe example steps in `examples/talks/tictactoe/` demonstrating integration with the Kronk SDK and the use of JSON Schema for structured output.

## Improvements
- **Context-Aware MaxTokens**: Refined `MaxTokens` handling so that if it is not explicitly provided in a request, it now defaults to the model's context window size instead of a fixed 4096.
- **Model Configuration**: Added `presence_penalty` support for `Qwen3.6-35B-A3B-UD-Q8_K_XL/AGENT`.

## Documentation
- **Refactored Examples**: Relocated and updated talk and workshop materials in `examples/talks/`.
- **Clarified Comments**: Updated documentation comments regarding `MaxTokens` and `DefMaxTokens` for better clarity and accuracy.
